AI Scripting Tutorial

Intro

In this tutorial, you will learn how to create your own AI script. This script is what will control your avatar throughout the game. The more complex you make it, the more interesting your avatar will become. By scrolling through this page, you will find all the necessary information to correctly write code for your avatar. At any time, you can press on the sidebar buttons to jump forward to any section of your choice. Once the tutorial is finished, close the tab and return to the Create Actor window to implement everything you've just learned.

How to Begin

For your AI script to function properly, you must write it in such a way that our program can understand it. Think of our program as a dog. Unfortunately, it only knows certain commands. If you ask it to do something that is not in its vocabulary, it will not understand and return an error. These commands must also be written in such a way so that they make sense syntatically. If you are continually getting error messages, be sure to refer back to this page to ensure that you are adhering to all the necessary syntax structure and format of your script as outlined in this help page. Much of the fun from this simulation will come from experimenting with many different styles of AI scripts!

How to Setup the Game

Begin by Clicking on "Start New Game"
You will be asked to enter a name for a new game
Close this page and now click on "Create Actor"
Modify this actor to act as you wish
Once actor is as desired, click on "Add Actor to Game"
Add this actor whichever game you'd like to see it in
Finally, click on "Load Game" to begin playing

Example script structure: if nearest(WATER) == MY_LOCATION then do walk(NORTH); done else do walk(direction(NORTH)); done endif Referring to the example above, take note of the following rules regarding script syntax:
  • All calls to an action end with a semicolon
  • 'do' and 'done' enclose all resulting actions from a conditional evaluation
  • Note that we start with a conditional 'if'. The second block of script evaluates if the above condition evaluates to true
  • 'then' must follow any conditional, to tell the script what to do if a conditional is fulfilled
  • All directions and cell attributes are in upper case
  • Adhering to the above structure, one can stack together multiple conditional statements to allow their AI to adapt to certain situations in the simulation

Action Words

The following words are actions that the actor can perform. They can be utilized in a variety of ways to help your avatar surive in the world. Each action will have an accompanying example in the scripting language to help better understand how it is used. These words will generally only occur within a do - done block.

Attack

Attack in the direction that your avatar is facing. Attacking another avatar will cause that avatar to lose health.

Example usage: if MY_ENERGY > 0 then do attack(); done endif

Drop

Takes an attribute, either FOOD or BLOCK, as an attribute and will drop the item, if possible, in the current location

Example usage: if HOLDING_FOOD then do drop(FOOD); done endif if HOLDING_BLOCK then do drop(BLOCK); done endif

Eat

If you are currently holding food, actor will eat, and thus regenerate health.

Example usage (combined): if MY_HEALTH < 10 then do eat(); done endif

Flee

Action performs the opposite of a list of directions to something.

Example usage: if nearest(DANGER) == MY_LOCATION then do flee(); done endif

Harvest

Takes a direction as a paremeter. Attempts to harvest in the direction that is given. This action is best used through the scavenge function. Because unless, given a direction in which a plant exists, nothing will occur.

Example usage: harvest(NORTH);

Pickup

Attempt to pickup an object in the cell next to the current location of the actor. If no object is present, actor will walk towards nearest block or walk in random direction depending on how far the said object is.

Example usage: if not HOLDING_BLOCK then do pickup(); done endif

Scavenge

Takes PLANT or FOOD as a parameter. Searches for the nearest food source. This is ultimately called to harvest a plant

Example usage: if MY_HEALTH == 10 then do scavenge(PLANT); done endif

Sleep

Puts your avatar to sleep, for a specified amount of time

Example usage: if MY_ENERGY < 50 then do sleep(); done endif

Walk

Takes in a list of directions as an argument and performs them. The list can also be only one direction.

Example usage: if true then do walk(direction(nearest(ROCK))); done endif OR walk(NORTH);

Actor Information

The following words are available to obtain characteristics of the current actor. They are useful to use as the 'conditionals' in your script. For example: 'if MY_HEALTH > 50'. The examples given will show their potential usage in conditionals and various situations

direction

Returns a list of directions to a specific coordinate. Accepts a tuple. Example calls: direction(nearest(WATER)), or direction (0 , 5). The first example will give you a list of directions to the nearest water element. The second will give you a list of directions to location (0,5) on the map.

Example usage (combined): direction(1,0) := [NORTH] direction(nearest(FOOD)) := [NORTH, WEST]

nearest()

Returns the coordinate of the nearest cell with the specified attribute.

Example usage in conditional and resulting action if nearest(WATER) == MY_LOCATION; walk(direction(nearest(DANGER)));

MY_DIRECTION

Gives the direction that the current actor is facing.

Example usage: if MY_DIRECTION == NORTH

HOLDING_FOOD

Returns 'True' if the current actor is holding food. Otherwise will return 'False'.

Example usage: if HOLDING_FOOD then do eat(); done endif

MY_HEALTH

Returns the current health of the actor

Example usage: if MY_HEALTH >= 40

MY_LOCATION

Returns the current location of the actor in the form of a 2-tuple of integers

Example usage: if nearest(WATER) == MY_LOCATION

SLEEPING

Returns 'True' if the actor is SLEEPING, otherwise will return 'False'

Example usage: if SLEEPING then do sleep(); done endif

HOLDING_BLOCK

Returns 'True' if the actor is holding a block, otherwise will return 'False'.

Example usage: if HOLDING_BLOCK then do drop(); done endif
